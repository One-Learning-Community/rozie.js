# Phase 07.3.1 Plan 06: Svelte snippets-merge for dynamic-name slot bridge (D-SV-16) Summary

## What Changed

### Task 1 — emitScript.ts wires the producer-side merge

Three additions to `packages/targets/svelte/src/emit/emitScript.ts`:

1. **`buildPropsInterfaceFields`**: when `ir.slots.length > 0`, appends `snippets?: Record<string, Snippet<[any]>>;` to the Props interface body (with provenance comment `Phase 07.3.1 D-SV-16`). The `Snippet` type already imports from `'svelte'` via the existing `collectSvelteImports` path because slots are present.

2. **`buildPropsDestructureEntries`**: when `ir.slots.length > 0`, renames each slot destructure entry from `header` to `header: __headerProp` (and `children` → `children: __childrenProp`, etc.), then appends `snippets` as a fresh destructure entry. When `ir.slots.length === 0`, the legacy no-rename / no-snippets shape is preserved verbatim — non-slotted components (Counter, SearchInput, leaf wrappers) are unaffected.

3. **`emitSlotDerivedMerges(ir)` (NEW helper)**: returns one `const ${key} = $derived(__${key}Prop ?? snippets?.${key});` line per slot. Returns `[]` when `ir.slots.length === 0`. Splice point is immediately after the destructure block, separated by a blank line.

4. **`emitPropsBlock`** updated to splice the merge block: `${interfaceBlock}\n\n${destructure}${mergeBlock}` where `mergeBlock` is `'\n\n' + mergeLines.join('\n')` when non-empty, else `''`.

### Task 2 — Unit test + snap re-blessings

**Unit test** `describe('snippets-merge (Phase 07.3.1 D-SV-16)', ...)` in `emitScript.test.ts` with **5** `it()` blocks (plan called for 3+, we added 5 to cover the threat-model precedence rule explicitly):

1. Props interface contains `snippets?: Record<string, Snippet<[any]>>;` when slots present (synthetic 2-slot IR via `.rozie` parser).
2. Destructure renames slot entries + appends `snippets` + emits the `$derived` merge line (single-slot IR).
3. T-07.3.1-14 precedence guard: `__headerProp ?? snippets?.header` (left-precedence). The statically-named consumer fill wins by construction.
4. Default-slot (`<slot />` without `name`) lowers to `children` and gets the same merge treatment.
5. Counter (no slots) byte-equivalent — no `snippets?:`, no `__\w+Prop`, no `$derived(__\w+Prop ?? snippets...)`.

**Snap fixtures** re-blessed via `vitest -u`:

- `packages/targets/svelte/fixtures/Modal.svelte.snap` — Modal has 3 slots (header/children/footer). Added `snippets?:` field, renamed 3 destructure entries, added 3 `$derived` merge lines.
- `packages/targets/svelte/fixtures/Modal.script.snap` — per-block fixture for Modal script; same diff.
- `packages/targets/svelte/fixtures/Dropdown.svelte.snap` — 2 slots (trigger/children). Same diff shape.
- `packages/targets/svelte/fixtures/TodoList.svelte.snap` — 3 slots (header/children/empty). Same diff shape.

Template snaps (`Modal.template.snap`, `TodoList.template.snap`, `Counter.template.snap`) were NOT updated because they cover only the template-region output — the slot identifiers (`header`, `children`, `footer`) still resolve to the same names in the rendered template after our rename (the `$derived` const re-binds the name).
