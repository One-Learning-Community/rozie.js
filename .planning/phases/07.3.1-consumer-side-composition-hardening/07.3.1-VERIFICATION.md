**CR-01 Svelte name collision (REVIEW critical finding):** `emitSlotDerivedMerges` emits `const ${key} = $derived(...)` without checking for collisions with `ir.state`/`ir.computed`/script-local names. For `Modal.svelte` and all current fixtures, no collision exists — `Modal` uses `header`/`children`/`footer` as slots and `backdropEl`/`dialogEl` etc. as state. The issue only bites when a component simultaneously declares a slot named `X` AND has `<data> X = ... </data>` — hypothetical in current dogfood but real risk for user components. REVIEW marks this Critical (follow-up fix needed); it does not prevent the current phase goal from passing.

**CR-03 Slot name injection (REVIEW critical finding):** Real risk for future user-authored components with non-identifier slot names. All 8 reference examples use valid JS identifier slot names (`header`, `footer`, `children`, `trigger`, etc.), so no current fixture is affected.
