## Code-Review Follow-Ups (advisory, deferred to 07.3.2)

The post-phase code-review surfaced 3 Critical + 6 Warning + 4 Info findings
(see 07.3.1-REVIEW.md). The 3 Critical findings are architectural concerns
that should be addressed in a follow-up phase 07.3.2:

- **CR-01**: Svelte `$derived` slot merge (Plan 06) emits top-level const
  declarations without collision detection vs `ir.state`/`ir.computed`/
  `ir.refs`/`ir.emits`. A component with a `children` data binding + a
  default slot produces SyntaxError. Fix: add collision check at lowerer
  or rename the destructured temp using a guaranteed-unique suffix.
- **CR-02**: Lit pre-seed (Plan 05) `nodeType !== 3` text-node guard on
  `this.children` is dead code — `this.children` is an HTMLCollection that
  only contains Elements (nodeType === 1) by definition. Bare-text default
  slot fills leave `_hasSlotDefault = false` forever. Fix: also walk
  `this.childNodes` for non-empty text nodes in the default-slot branch.
- **CR-03**: Slot names and prop names are interpolated into emitted CSS
  selectors, HTML attribute values, JS template literals, and field
  identifiers throughout the Lit + Svelte emitters with no validation. An
  adversarial slot name like `2items`, `foo-bar`, or `foo"]bar` silently
  produces broken output. Fix: add `validateIdentifierName` check at IR
  lowering time, emit ROZ959 (TBD) diagnostic.

These are not regressions introduced by Phase 07.3.1 — most are pre-existing
patterns inherited from prior phases. Phase 07.3.1 just made them more
visible by adding new identifier interpolation sites.
