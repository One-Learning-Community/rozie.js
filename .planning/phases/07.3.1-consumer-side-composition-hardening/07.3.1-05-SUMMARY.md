# Phase 07.3.1 Plan 05: Lit pre-seed _hasSlot<X> from light DOM in connectedCallback (D-LIT-15) Summary

**Lit producer now inspects light-DOM children inside `connectedCallback()` BEFORE `super.connectedCallback()` and pre-seeds `_hasSlot<X>` state — closes the chicken-and-egg deadlock where conditionally-rendered slot wrappers prevented `<slot>` elements from existing, which starved `@queryAssignedElements`, which kept `_hasSlot<X>` permanently false.**

## Accomplishments

- **D-LIT-15 closed at the emitter layer.** `_hasSlot<X>` `@state()` fields are seeded from `this.children` synchronously inside `connectedCallback()` before `super.connectedCallback()` so the first render cycle reflects actual consumer fill presence. This unblocks Modal 1 (static scoped `#header` fill, no `title` prop) and Modal 2 (dynamic-name `#[slotName]` fill resolving to 'header') in the `examples/consumers/lit-vanilla-demo` close-spec dogfood — both previously rendered with no header forever because the chicken-and-egg deadlock kept `_hasSlotHeader` `false`.
- **5 new regression tests** lock the contract at the @rozie/target-lit unit level — future emitter refactors that drop the pre-seed will be caught long before the visual-regression suite. The Modal-fixture test slices the emitted `connectedCallback()` body via regex and asserts pre-seed `indexOf` < `super.connectedCallback();` `indexOf`, locking the ordering invariant.
- **4 snap fixtures re-blessed** with surgical, mechanically-uniform diffs: one provenance comment plus N pre-seed assignment lines per fixture, spliced between `connectedCallback() {` and `super.connectedCallback();`. Zero unrelated drift.

## Decisions Made

- **Splice site is `composeClassBody` in `emitLit.ts`, not `shell.ts`.** The plan named `shell.ts` for the splice but `shell.ts` only composes imports, decorator, and class frame. The class body — including the existing `super.connectedCallback();` template literal — is assembled in `composeClassBody` inside `emitLit.ts`. Modifying the actual emit site is the only correct interpretation. Documented in the Task 1 commit body and as deviation below.
- **connectedCallback gating relaxed from `hasListenerWiring` to `hasListenerWiring || hasSlotPreSeed`.** Defensive against future emitter changes that might remove slotchange wiring. For the current code path this is a no-op (any component with slots has slotchange wiring → has listener wiring), so no snap drift on no-slot fixtures (Counter/SearchInput/TreeNode/CardHeader).
- **Pre-seed runs on every connect, not just first.** The plan's mechanism inherently runs on every `connectedCallback()` invocation — re-mount cycles also re-seed. This is correct behaviour for any disconnect/reconnect where the consumer's light-DOM children may have changed.
- **Default-slot text-node guard uses `(el.textContent?.trim().length ?? 0) > 0`.** Web Components spec treats whitespace-only text nodes as ignored for slot fill detection. Without this guard, a `<rozie-modal>` containing only formatting whitespace between tags would falsely report `_hasSlotDefault = true`.
