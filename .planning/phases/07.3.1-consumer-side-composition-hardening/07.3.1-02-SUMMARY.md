## Decisions Made

- **Object-payload + shorthand collapse:** Output uses `{ close }` not `{ close: close }` when IR arg `name` equals the rewritten expression string. Keeps emit idiomatic and matches what a Vue developer would expect (Vue's `defineSlots<{ header(p: { close: any }): any }>` convention). When the expression is non-trivial (e.g., `:total="$props.items.length"` → `total: items.length`, or `:toggle="() => toggle(item.id)"` → `toggle: () => toggle(item.id)`) the full `key: expr` form emits.
- **Zero-arg short-circuit:** When `node.args.length === 0`, `argList` is the empty string, producing `{@render footer?.()}` (no empty object literal). This preserves the existing zero-arg snippet behavior and avoids a gratuitous `({ })`.
- **Single-tuple object-literal type:** Per RESEARCH Open Question #1 recommendation, chose `Snippet<[{ open: any; toggle: any }]>` over `Snippet<[any]>`. Rationale: TypeScript can now catch consumer-side destructure mismatch (a consumer that tries `{#snippet trigger({ wrongKey })}` will surface as a TS error rather than silent `undefined`). Aligns with Vue's `defineSlots` and React's prop-style render-prop typing.
- **Modal.script.snap inclusion:** PATTERNS.md listed 5 snap fixtures (Modal/Dropdown/TodoList × .svelte.snap + Modal/TodoList × .template.snap). The actual vitest -u run regenerated 6 — Modal.script.snap was the +1, capturing the type-signature region of the script-block-only snapshot test. Atomically committed alongside the listed 5; no scope expansion (same emit-output content, different test viewpoint).
