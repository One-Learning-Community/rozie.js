## Cross-Framework Consumption Gotchas Surfaced

- **Vue**: `:value` on a custom element falls back to setting an attribute (not a property) since the Vue template compiler can't statically detect the element's property list. The Counter's attributeChangedCallback wires this correctly via `notifyAttributeChange`. Documented as expected behavior.
- **React 19**: Customelement support works as advertised — `onvalue-change` (lowercase-dashed) routes to the property handler. The `value={5}` JSX attribute is passed as a PROPERTY (not attribute) at first paint, which hits a v1 emitter gap: the property setter goes through the controllable's `write()` but does NOT call `host.requestUpdate()` or update the `_value_attr` mirror, so the internal display does not reflect the initial controlled value. The fundamental click → event → React state flow works.
- **Svelte 5**: Event-attribute form (`onvalue-change={handler}`) works. `bind:value` would not — confirmed by the route NOT using it per Pitfall 8.
- **Angular standalone**: `CUSTOM_ELEMENTS_SCHEMA` narrowly scoped to the lit-interop component works at compile time (Vite consumed the imports without error). Runtime test gated by AnalogJS+Vite blocker.
