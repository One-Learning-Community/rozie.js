## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 — Bug fix] `[^]` body rule emits per-character tokens, not "single greedy run"**
- **Found during:** Task 1 (lexer regeneration + snapshot verification)
- **Issue:** Plan + RESEARCH Pattern 1 verbatim specify `<IN_TEMPLATE_BODY> { ^"</template>" { ... } [^] { return TEMPLATE_BODY; } }`. The done-criteria claim "Snapshots show TEMPLATE_BODY runs spanning entire template bodies in all 7 retained fixtures." But `[^]` is a single-character class in JFlex — it produces 1-char tokens per character (verified empirically: Counter.rozie.txt regenerated as 429 lines of per-char TEMPLATE_BODY tokens). Cannot use `[^]+` greedy because it would conflict with the line-anchored close (longest-match would let `[^]+` eat through the entire rest of the file).
- **Fix:** Replaced with the canonical IN_SCRIPT_BODY shape (`[^<]+` + plain `<`) — JFlex longest-match still pins `^"</template>"` (11 chars) over a stray `<` (1 char) at column-0 positions, so the close rule fires correctly. Inner `<template>` tags inside the body live in the greedy run interleaved with `<` chars. This pattern matches PATTERNS § "Secondary analog" line 121–128 verbatim — the file's own `IN_SCRIPT_BODY` is the cited template.
- **Files modified:** tools/intellij-plugin/src/main/jflex/Rozie.flex
- **Verification:** Regenerated snapshots show Counter.rozie.txt at 70 lines (down from 429), template bodies as runs of TEMPLATE_BODY tokens. `edge-nested-template.rozie.txt` correctly fires `TEMPLATE_CLOSE_TAG` only on the outer column-0 `</template>` and includes the inner indented `</template>` as a TEMPLATE_BODY fragment.
- **Committed in:** 146f39f (Task 1 commit)
