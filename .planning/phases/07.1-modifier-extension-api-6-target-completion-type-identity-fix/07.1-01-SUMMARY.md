## Decisions Made

- **Self-reference = tsconfig `paths` ONLY.** The plan offered two sub-approaches (A1 tsconfig `paths`, A2 `workspace:*` self-dep) and recommended combining `paths` with "whatever minimal package.json change makes runtime/bundled resolution correct." In practice the `paths` mapping alone yields green `tsc --noEmit` AND green `turbo run typecheck` (verified via scratch self-import). The `workspace:*` self-dep was tried but **broke turbo's task graph** (`@rozie/core#build depends on itself`) and was removed — see Deviations. No runtime self-reference is needed in v1: `compile.ts` keeps its relative target imports (Option A scope, untouched), and target packages resolve `@rozie/core` through their own normal workspace deps. **Plan 02's established resolution contract: tsconfig `paths` in `packages/core/tsconfig.json` only — do NOT add a `workspace:*` self-dependency.**
- `SolidEmissionDescriptor` mirrors `ReactEmissionDescriptor`'s three-discriminant shape (native/helper/inlineGuard); `LitEmissionDescriptor` mirrors `Svelte`/`Angular`. `helperName` unions match the verified `@rozie/runtime-{solid,lit}` exports (read before defining — RESEARCH Pitfall 4 mitigation).
- Per-builtin test coverage for `solid()`/`lit()` — there is no aggregate `svelte-hooks`/`angular-hooks` test in core (only `vue-hooks`/`react-hooks`), so per-file extensions suffice per the plan's conditional instruction.
