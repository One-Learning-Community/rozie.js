## Decisions Made

- **Per-package `paths` mapping is the resolution mechanism.** Plan 01 added a `paths` mapping to *core's own* tsconfig. This plan found that is necessary but not sufficient: every CONSUMER package that BOTH imports `@rozie/core` AND relative-imports `core/src/*` (all 6 targets, unplugin, cli) ALSO needs the mapping, because within that package's `tsc` run the `@rozie/core` specifier otherwise resolves through `package.json` `exports` to `core/dist/index.d.mts` while the relative imports resolve to `core/src/*` — two `ModifierRegistry` nominal identities, `TS2345`. The mapping pins both to `core/src/index.ts`.
- **Lit handler param re-typing.** Plan 01 wired the builtins' `lit()` from the reviewed `svelte()`/`angular()` strings, which are cast-free (`if (e.key !== 'Escape')`) because Svelte/Angular type their handler params with a precise DOM event interface. Lit's old hand-rolled emitter used `(e: Event)` + per-guard `(e as KeyboardEvent)` casts. To consume the registry-driven cast-free `inlineGuard` codes without a typecheck error, the Lit emitter now types the handler param via an `eventTypeFor` helper (mirroring Svelte). This is a structural-equivalence change — same key check, same early-return — affecting 3 lit fixtures, rebaselined.
- **cli in scope, babel-plugin not.** `@rozie/cli` typechecks the target `src/*.ts` files transitively (via `compile()`'s emitter imports) and hit the identity split — added the `paths` mapping. `@rozie/babel-plugin` did NOT fail the gate without the mapping (its transitive walk does not cross the `ModifierRegistry` type boundary), and adding the mapping there actively broke its build because `babel-plugin`'s tsconfig sets `rootDir: ./src` and the mapping pulls `core/src/*` + `targets/*/src/*` files outside that rootDir (`TS6059`). Left babel-plugin untouched.

### Snapshot Changes (plan-caused, justified)

**Lit `<listeners>`-block key-filter handler typing — 5 fixtures rebaselined.**
Task 3's registry-dispatch rewrite consumes the builtins' `lit()` `inlineGuard` codes, which are cast-free (`if (e.key !== 'Escape') return;`). The old hand-rolled Lit emitter typed handlers `(e: Event)` and inserted per-guard `(e as KeyboardEvent)` casts.
