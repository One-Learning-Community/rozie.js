# Phase 07.3 Plan 07.3-06: React consumer-side two-way binding emit Summary

D-01 React idiom â€” `<Modal r-model:open="$data.open1">` lowers to the JSX attribute pair `open={open1} onOpenChange={setOpen1}`, replacing the Wave 2 throw-stub in `emitNonClassAttribute` with a real emit branch backed by a new pure helper `resolveTwoWayTarget` that resolves the writable LHS to its `{ local, setter }` JSX-name pair.

## What Was Built

### New: `resolveTwoWayTarget` helper

**File:** `packages/targets/react/src/emit/resolveTwoWayTarget.ts` (NEW)

Pure function: `resolveTwoWayTarget(expr, ir) â†’ { local, setter }`. Three branches:

1. **`$data.X`** (useState-backed state) â†’ `{ local: 'X', setter: 'set' + capitalize(X) }`. Matches the convention emitRModel.ts:67 uses for the form-input `r-model` case, so consumer-side and producer-side setter naming can never drift.

2. **`$props.X`** with `model:true` (forwarding pattern, D-03 permissive) â†’ `{ local: 'X', setter: 'on' + capitalize(X) + 'Change' }`. Matches the convention emitPropsInterface.ts:92 uses to synthesise the model-prop callback field, so consumer code can forward the upstream callback directly without renaming glue.

3. **`$data.X.Y.Z`** (deep chain, D-03 permissive) â†’ `{ local: rewritten-chain-text, setter: '(v) => { <local> = v; }' }`. v1 fallback per RESEARCH Â§"isWritableLValue helper" â€” consumers are expected to flatten to top-level `$data.X` for clarity; this branch exists for completeness so the validator's permissive D-03 surface compiles end-to-end.

The helper deliberately throws on unreachable cases (anything the IR-time validator should have rejected via ROZ951). This surfaces a regression in `isWritableLValue` immediately rather than silently producing malformed JSX.

### Modified: `emitTemplateAttribute.ts` twoWayBinding branch

**File:** `packages/targets/react/src/emit/emitTemplateAttribute.ts`

Replaced the Wave 2 throw-stub in `emitNonClassAttribute` (~ lines 588-594 of the prior shape) with the real emit. New code shape:

```ts
if (attr.kind === 'twoWayBinding') {
  const { local, setter } = resolveTwoWayTarget(attr.expression, ctx.ir);
  const propName = colonPropToJsxName(attr.name);
  const eventProp = `on${propName.charAt(0).toUpperCase()}${propName.slice(1)}Change`;
  return {
    jsx: `${propName}={${local}} ${eventProp}={${setter}}`,
    diagnostics,
  };
}
```

camelCase propNames pass through `colonPropToJsxName` unchanged (no hyphens / aria-/data- to mangle), so `r-model:closeOnEscape="..."` correctly emits `closeOnEscape={...} onCloseOnEscapeChange={...}`.

The class-merge stub (~ line 325) stays as `throw` because `twoWayBinding` cannot legally participate in className composition (it would never be bucketed under `name === 'class'`); the throw surfaces a regression in the lowerer if one ever sneaks in.
