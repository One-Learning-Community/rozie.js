# Phase 07.3 Plan 04: Svelte Target Consumer-Side Two-Way Binding Emit

## One-liner

Svelte 5 runes-mode `bind:propName={<expr>}` emit for `<Producer r-model:propName="expr"/>` against producer's `$bindable()`.

## What was built

Wave 3 Plan 04 (Svelte leg of the 6-target consumer-side two-way binding fan-out). Replaces the Wave 2 throw-stub in `packages/targets/svelte/src/emit/emitTemplateAttribute.ts` `emitSingleAttr` with the Svelte 5 runes-mode two-way binding form:

```
<Modal r-model:open="$data.x" />   →   <Modal bind:open={x}>
```

The producer-side `$bindable(...)` machinery (TWO-WAY-02, already shipped) wires writes from the bind: site straight back through the producer's rune. This plan is purely the consumer-side template-emit branch.

Routes through the existing `rewriteTemplateExpression(attr.expression, ir)` helper so `$data.x` strips to `x`, `$props.active` (when consumer's own prop is declared `model: true`) strips to `active`, and ref/slot identifiers Just Work for free. No new rewrite paths needed.

### Code change

```ts
if (attr.kind === 'twoWayBinding') {
  const expr = rewriteTemplateExpression(attr.expression, ir);
  return `bind:${attr.name}={${expr}}`;
}
```

The sibling `attrToArraySegment` stub stays as a `throw` — `twoWayBinding` never participates in the class/style array-merge context (validator rejects it upstream).

### [Test-fixup] hard-negative regex needed negative lookbehind

- **Found during:** Task 2 (after GREEN run).
- **Issue:** Initial hard-negative `/[^d]open=\{x\}/` and follow-up `/:open=/` both falsely matched `bind:open={x}` itself (the `:` in `bind:` is `[^d]` and the substring `:open=` is also literally present).
- **Fix:** Switched to `/(?<!bind):open=/` and `/(?<!bind:)open=\{x\}/` — negative lookbehind disambiguates the Vue-style `:open=` shape from the legitimate Svelte `bind:open=` shape.
- **Files modified:** `packages/targets/svelte/tests/emit-two-way.test.ts`.
- **Commit:** Folded into the feat commit (`fb87296`) so the test as committed reflects the green state.
