### Auto-fixed Issues

**1. [Rule 3 — Blocking issue] Slot-matrix fixture path adjusted from single-file to directory form**
- **Found during:** Task 2 (slot-matrix fixture creation)
- **Issue:** Plan frontmatter and acceptance criteria specified
  `tests/slot-matrix/fixtures/consumer-two-way-fill.rozie` (single file at the
  fixtures root). The slot-matrix runner at `tests/slot-matrix/slot-matrix.test.ts`
  enumerates fixture directories via `readdirSync(FIXTURES_DIR, { withFileTypes: true })
  .filter((d) => d.isDirectory())` — a single `.rozie` file at the root is
  silently invisible to the matrix. Creating it as plan-specified would have
  produced an inert artifact that Wave 4's snapshot generation could not find.
- **Fix:** Created the fixture as a directory `tests/slot-matrix/fixtures/consumer-two-way-fill/`
  with `input.rozie` (consumer with `<Producer r-model:open="$data.outerOpen">`
  + a `<template #footer="{ close }">` body that nests another `<Inner r-model:open=…>`)
  + `producer.rozie` (sibling, declares `open: { type: Boolean, model: true }`),
  matching the format of every existing `consumer-*-fill` fixture (e.g.
  `consumer-scoped-fill/`, `consumer-named-fill/`).
- **Files modified:** `tests/slot-matrix/fixtures/consumer-two-way-fill/{input,producer}.rozie`
