# Plan 07.3-02 — Compiler-core foundation for consumer-side two-way binding

## What was built

Wave 2 lands the compiler-side foundation that Wave 3's 6 per-target emitters compile against.

**New IR variant (D-01 keystone):** `AttributeBinding` union gains a fourth
discriminated member `{ kind: 'twoWayBinding', name: string, expression: Expression, deps: SignalRef[], sourceLoc: SourceLoc }`. Re-exported from the `@rozie/core` barrel per Phase 07.1 self-reference pattern.

**Lowerer branch:** `packages/core/src/ir/lowerers/lowerTemplate.ts` recognises
template attributes whose name matches `r-model:<propName>` and lowers them to
the new `twoWayBinding` IR variant. Empty argument (`r-model:=`) emits ROZ950 at lower time.

**Shared LHS helper (D-03 permissive rule):** `packages/core/src/semantic/lvalue.ts`
exposes `isWritableLValue(expr, ir)` covering `$data.x`, `$data.x.y.z`, and
`$props.x` when the consumer's own `<props>` declaration has `model: true`.
Rejects literals, ternaries, calls, `$computed` refs.

**IR-time validator (D-02):** `packages/core/src/ir/validateTwoWayBindings.ts`
mirrors `threadParamTypes` line-for-line — same walker, same `IRCache +
ProducerResolver` lookup, same silent-degrade on `producerIR === null`. Emits
ROZ949 dual-frame (consumer site + producer decl), ROZ950 shape errors,
ROZ951 LHS errors. Priority: ROZ950 > ROZ951 > ROZ949.

**Pipeline wiring:** `compile.ts` invokes `validateTwoWayBindings` after
`threadParamTypes`. `packages/unplugin/src/transform.ts` adds a parallel
`validateTwoWayBindingsForPipeline` helper and calls it after each of the
7 `threadParamTypesForPipeline` sites (lines 687/751/815/881/946/1015/1311).

**Exhaustiveness stubs in 5 target emitters:** Vue / Svelte / Angular / React /
Solid `emitTemplateAttribute.ts` each add an early `if (attr.kind === 'twoWayBinding')`
branch that throws a clear "Phase 07.3 Wave 3 Plan 07.3-NN" error. Each Wave 3
plan replaces its target's stub with the real consumer-side two-way emit.
(Lit already had proper exhaustiveness.)
