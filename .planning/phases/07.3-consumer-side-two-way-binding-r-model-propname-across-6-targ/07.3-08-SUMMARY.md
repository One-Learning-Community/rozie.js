# Phase 07.3 Plan 08: Lit target consumer-side two-way binding emit

## One-liner

Lit consumer-side `r-model:propName="$data.x"` now emits the
`.<prop>=${value} @<kebab(prop)>-change=${(e: CustomEvent) => { setter = e.detail; }}`
html-template pair, wiring INTO the producer-side
`createLitControllableProperty` CustomEvent contract locked in Phase 06.4.

## What was built

**`packages/targets/lit/src/emit/resolveLitSetterText.ts` (new, 159 LOC):**

- `resolveLitSetterText(expr, ir)` resolves a writable LHS expression to its
  Lit setter-target text:
  - `$data.X` (top-level signal) â†’ `this._X.value` (preact-signals signal
    assignment idiom)
  - `$data.X.Y.Z` (deep chain) â†’ `this._X.value.Y.Z` (root rewritten;
    subsegments preserved verbatim)
  - `$props.X` (consumer's own `model: true` prop â€” the forwarding /
    wrapper-component pattern) â†’ `this.X` (Lit `@property` setter â€” invokes
    the controllable's `write()`)
  - Generic-fallback: `@babel/generator` over the cloned expression. The
    IR-time validator (`validateTwoWayBindings`, ROZ951 from Plan 07.3-02)
    is the primary gate against non-writable LHS; this fallback emits
    something debuggable rather than throwing at emit time.

- `kebabize` is **re-exported from `emitDecorator.ts`** (the existing
  producer-side `toKebabCase`) â€” NOT a parallel hand-rolled algorithm.
  Rationale: the consumer's `@<eventName>` listener MUST byte-equal the
  producer's `dispatchEvent(new CustomEvent('<eventName>', ...))`. Sharing
  the algorithm by re-export makes that invariant a structural property
  (cannot drift) rather than a documented convention.

**`packages/targets/lit/src/emit/emitTemplateAttribute.ts` (+15 LOC):**

- Adds `if (attr.kind === 'twoWayBinding')` branch BEFORE the existing
  static/binding/interpolated cascade. Returns the literal Lit html``
  template-fragment string:
  ```ts
  `.${attr.name}=\${${valueExpr}} @${eventName}=\${(e: CustomEvent) => { ${setterText} = e.detail; }}`
  ```
  where `valueExpr = rewriteTemplateExpression(attr.expression, ir)`,
  `setterText = resolveLitSetterText(attr.expression, ir)`,
  `eventName = ${kebabize(attr.name)}-change`.
- Leading comment documents the D-01 producer contract and the
  `(e: CustomEvent)` landmine guard (Lit's default `@event` arg type is
  `Event`, which does not expose `.detail` â€” untyped `(e)` would compile
  but fail consumer-side TS).

## Deviations from Plan

### [Rule 1 â€” Bug] kebabize algorithm aligned with producer (was: naive /[A-Z]/g rule)

**Found during:** Task 1 (TDD GREEN â€” `kebabize('aBC')` test case failed)

**Issue:** The plan's `<action>` step 1 specified
`kebabize converts camelCase â†’ kebab-case (regex /[A-Z]/g â†’ '-' + lowercase);
test cases: open â†’ open, closeOnEscape â†’ close-on-escape, aBC â†’ a-b-c`. The
naive `/[A-Z]/g` rule produces `aBC â†’ a-b-c` AND `CloseOnEscape â†’ -close-on-escape`
(leading hyphen artifact), and â€” most critically â€” **diverges from the
producer-side `toKebabCase`** (`packages/targets/lit/src/emit/emitDecorator.ts:15`)
which only hyphenates at `[a-z0-9][A-Z]` boundaries. The producer emits its
CustomEvent name via `toKebabCase` (see emitScript.ts:172:
`eventName = \`\${toKebabCase(prop.name)}-change\``), so a consumer
`@<naive-kebab>-change` listener would NEVER fire for any prop name with
adjacent capitals because the names would not match byte-for-byte.

**Fix:** `kebabize` is **re-exported from `emitDecorator.ts`** (`export
{ toKebabCase as kebabize } from './emitDecorator.js'`). Test case
`aBC â†’ a-bc` adjusted accordingly (with explanatory comment in the test).
For all realistic prop names (lowerCamelCase identifiers with no adjacent
capitals â€” `open`, `closeOnEscape`, `closeOnBackdrop`), both algorithms
produce identical output, so the user-visible behavior in production code
is unchanged. Re-export is structurally superior to duplication because the
byte-equal contract becomes a property of the code (cannot drift) rather
than a hope.

**Files modified:** `packages/targets/lit/src/emit/resolveLitSetterText.ts`
(re-export instead of hand-rolled algorithm),
`packages/targets/lit/src/__tests__/emit-two-way.test.ts` (assertion +
comment).

**Commit:** `5b0f48a` (GREEN â€” same commit as the implementation).
