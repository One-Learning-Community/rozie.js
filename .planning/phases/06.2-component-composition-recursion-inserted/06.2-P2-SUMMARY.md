## Self-reference handling per target

| Target | Mechanism | Where it lives | Why this is the canonical idiom |
|--------|-----------|----------------|----------------------------------|
| Vue | `defineOptions({ name: 'Self' })` macro inside `<script setup>` | shell prelude | Vue's filename-based auto-name doesn't reliably propagate through path-virtual schemes (Pitfall 2 — always emit) |
| React | named-function declaration `export default function Self(...)` (already in shell at line 201) | unchanged from existing emitter | function declarations are hoisted within their containing scope → `<Self ...>` JSX inside the body just works (Pitfall 7) |
| Svelte | `import Self from './Self.svelte';` at top of `<script>` | emitSvelte synthesizes if outer name not already in components table | Svelte 5 self-import idiom per CONTEXT D-117 UPDATE 2026-05-07 — modern Svelte 5 path; NOT legacy `<svelte:self>` (Pitfall 4) |
| Angular | `imports: [forwardRef(() => Self)]` inside `@Component({...})` + `forwardRef` in `@angular/core` import line | emitDecorator + AngularImportCollector | Angular requires forward-ref wrapper for class-self-reference in standalone-component imports[] (Pitfall 5) |

## Self-entry skip logic

**React + Angular:** filter out `IRComponent.components` entries whose `localName === ir.name`. Rationale:

- React's `import { Self } from './Self'` would shadow the enclosing function declaration in the same file.
- Angular's class is in scope of its own `@Component({...})` decorator — a top-of-file `import { Self } from './Self'` would create a circular import resolving to itself.

**Svelte:** opposite logic — synthesize the self-import line when `tagKind: 'self'` is present AND the outer name isn't already in the components table. The self-import idiom IS the binding mechanism.

**Vue:** redundant import is harmless — Vue's setup-scope is idempotent. Verified: `TreeNode.vue.snap` contains both `import TreeNode from './TreeNode.vue';` AND `defineOptions({ name: 'TreeNode' })`. Both work; either alone would also work.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Defensive `ir.components ?? []` guard in all 4 emitters**

- **Found during:** Vue Task — running full vue test suite revealed 5 generics tests failing with `Cannot read properties of undefined (reading 'map')`.
- **Issue:** Pre-P1 hand-rolled minimal IRs in `generics.test.ts` (and similar) construct `IRComponent` literals without the `components` field. The TS type now requires it, but those tests bypass the type system.
- **Fix:** Added `const components = ir.components ?? [];` before iteration in all 4 emitters (`emitVue.ts`, `emitReact.ts`, `emitSvelte.ts`, `emitAngular.ts`). Each emitter consumes `components` instead of `ir.components` for the import-line synthesis + filter.
- **Commits:** 7913ffd (vue), 3f4384a (react), ea05122 (svelte), 0b41308 (angular).

**2. [Rule 2 - Critical functionality] Angular composition tags need kebab-case selector mapping**

- **Found during:** Angular Task — initial emit produced `<TreeNode>` and `<CardHeader>` verbatim in template strings.
- **Issue:** Angular standalone components match by `selector` (kebab-case `rozie-tree-node`, `rozie-card-header`). PascalCase template tags would not actually match the registered selectors at runtime — the `<Counter>` template tag would render as an unknown HTML element rather than the `rozie-counter` standalone component.
- **Fix:** Added `resolveAngularTagName(node)` helper at `emitTemplateNode.ts` — maps `tagKind: 'component' | 'self'` to `rozie-${toKebabCase(node.tagName)}`; HTML tags emit verbatim. Imported `toKebabCase` from `emitDecorator.ts` (single source of truth).
- **Files modified:** `packages/targets/angular/src/emit/emitTemplateNode.ts`.
- **Commit:** 0b41308.
- **Verified:** TreeNode.angular.ts.snap renders `<rozie-tree-node [node]="$props.children"></rozie-tree-node>` matching the `selector: 'rozie-tree-node'` decorator field.

**3. [Rule 3 - Blocking] Test files placed in `src/__tests__/` not plan's path**

- **Found during:** Task 0 — the plan listed `packages/core/src/codegen/rewriteRozieImport.test.ts` (sibling test) but the project's `vitest.config.ts` glob pattern is `src/**/__tests__/*.test.ts`.
- **Fix:** Authored at `packages/core/src/codegen/__tests__/rewriteRozieImport.test.ts` to match existing project convention. Same applies to all 4 per-target composition tests (each in `src/__tests__/composition.test.ts`).
