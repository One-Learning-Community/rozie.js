## Decisions Made

See `decisions` block in frontmatter for full list. Key decisions:

1. **Separate rewriteTemplateExpression.test.ts file** (not extending rewriteScript.test.ts) — distinct functions, cleaner parallel test surface.
2. **t.parenthesizedExpression builder** preferred over `.extra={parenthesized:true}` fallback — builder verified present in @babel/types 7.29.x; cleaner code.
3. **jsescOption: { quotes: 'single' }** added to rewriteTemplateExpression's GEN_OPTS so synthesized `['header']` key matches emitSlotInvocation's single-quote style — Modal.tsx is internally consistent.
4. **Selector fix in modal-consumer-close.spec.ts** — out-of-plan but required to deliver Plan 08's success criterion (Rule 1 bug); literal `.X` selector failed React's CSS-Modules-hashed class.
5. **Local Playwright verification** instead of pinned Docker — text/DOM-presence assertions are platform-independent; Docker round-trip cost not justified for non-pixel assertions. Documented under "Environment Caveat" in VERIFICATION-NOTE.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 — Blocking] `pnpm install` populated worktree node_modules at agent startup**

- **Found during:** First `pnpm --filter @rozie/target-react test` invocation at agent startup.
- **Issue:** Worktree spawned without node_modules (Claude Code's `isolation="worktree"` default). Test runner errored with "Failed to resolve import '@rozie/runtime-react'".
- **Fix:** `CI=true pnpm install --frozen-lockfile --prefer-offline` (~14s) + `pnpm turbo run build --filter @rozie/runtime-react` to populate the workspace dist for `.test.tsx` integration tests that import the compiled runtime artifact.
- **Files modified:** none committed (node_modules in .gitignore).
- **Why Rule 3:** Test gate verification (the RED step) cannot run without populated node_modules. Standard workspace install — no Rule-3 package-install exclusion violated (no new packages added).

**2. [Rule 2 — Missing critical functionality] Added new it() block in emitTemplate.test.ts for the r-if guard merge**

- **Found during:** Task 1 GREEN — after snapshot auto-regen, realized the integration layer (lower → emit pipeline) had no direct assertion that `r-if="$slots.header"` lowers to the merged guard. Without it, a future regression in emitTemplateAttribute or a wrapper-emitter could silently revert the guard merge while the rewriter-layer unit tests still pass.
- **Issue:** The plan's Task 2 step 3 mentioned "update emitTemplate.test.ts" but only as an OPTIONAL nudge — the existing Test 4 already covers the invocation site. The GUARD-site contract needed its own assertion to gate future drift at the integration layer.
- **Fix:** Added new Test 5 in `Slot lowering — Plan 04-03 Task 2` describe block. Asserts the r-if guard shape `props.title || (props.renderHeader ?? props.slots?.['header'])` end-to-end through the lower+emit pipeline.
- **Files modified:** packages/targets/react/src/__tests__/emitTemplate.test.ts (+27 lines).
- **Committed in:** 25056e2 (folded into the GREEN commit since the test reflects the same contract).

**3. [Rule 1 — Bug] Fixed D-04 Modal 2 dynamic-fill Playwright selector**

- **Found during:** Task 3 verification — `react: Modal 2 dynamic-fill` FAILED even after Plan 08's guard merge made the `<header>` render. Playwright error context showed the DOM had a banner with the text "Dynamic header via slotName" but the locator `.dynamic-fill` matched 0 elements.
- **Issue:** React's CSS-Modules pipeline hashes the class to `_dynamic-fill_o9ucb_3`. The literal `.dynamic-fill` Playwright CSS selector requires a SEPARATE class named `dynamic-fill` (it doesn't match substrings). The original spec author's comment "unaffected by CSS Modules hashing for React/Solid" was wrong about CSS Modules semantics.
- **Fix:** Switched to `page.locator('[class*="dynamic-fill"]')` — CSS attribute substring selector that matches both unhashed (vue/svelte/lit/angular/solid) and CSS-Modules-hashed (React) classes.
- **Files modified:** tests/visual-regression/specs/modal-consumer-close.spec.ts.
- **Committed in:** af945a9 (separate commit so the test-instrumentation fix is auditable independent of the producer-source fix).
- **Why Rule 1 (Bug) not Rule 4 (Architectural):** The test as written didn't match the runtime DOM reality. This is a test-instrument bug, not an architectural decision. Fixing it is required to deliver the plan's primary success criterion (PASS Playwright assertion).
