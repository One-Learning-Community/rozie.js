## Blockers

### B-07.3.2-05-01 — Angular emitter backtick-escape bug breaks VR sub-build

**Severity:** HIGH (blocks SC#5 closure; blocks Phase 07.3.2-06 HUMAN-UAT readiness)
**Origin:** d77ffeb (`feat(examples): add Table.rozie producer + TableDemo consumer`, 2026-05-17 00:39)
**NOT caused by:** Plans 07.3.2-01/02/03/04 emitter changes (those are slots-merge intake additions only; this bug is in `emitDecorator.ts:156`, in code unchanged since Phase 5)

**Symptom:**
- **On macOS host:** `ROZIE_TARGET=angular pnpm exec vite build --config vite.config.ts` (run from `tests/visual-regression/`) fails fast (~900ms) with:
  ```
  [vite:esbuild] Transform failed with 1 error:
  examples/demos/TableDemo.rozie.ts:17:20: ERROR: Expected "}" but found "badge"
  ```
- **In pinned Playwright Docker container (`mcr.microsoft.com/playwright:v1.60.0-jammy`):** the same Angular sub-build hangs in `futex_wait` for 30+ minutes with vite child consuming only ~3 seconds of CPU. Verified via `cat /proc/<vite-pid>/wchan` → `futex_wait`. Other targets (vue, react, svelte) complete normally in <1s each.

**Root cause:** `packages/targets/angular/src/emit/emitDecorator.ts:156` wraps the compiled Angular template body in a JS template literal:
```js
lines.push(`  template: \``);
lines.push(indentedTemplate);
lines.push(`  \`,`);
```
…with NO escaping of backticks inside `indentedTemplate`. When `examples/demos/TableDemo.rozie` (added 2026-05-17 in d77ffeb) uses `\`badge badge-\${value}\`` Angular template syntax inside a `<span [class]="...">`, the emitted `.rozie.ts` disk-cache file looks like:
```ts
template: `
  …
  <span [class]="`badge badge-${value}`">{{ value }}</span>
  …
`
```
The inner backtick at column 20 of line 17 closes the OUTER template literal, leaving the rest of the file unparseable.

**Why this wasn't caught earlier:**
- No existing fixture (Counter/Dropdown/TodoList/Modal/etc.) uses backticks inside its `<template>` block; they all use double-quoted string concatenation or `{{ }}` interpolation. TableDemo.rozie is the first canonical example to do so.
- The cleanup function `cleanupCrossTreeAngularArtifacts()` in `tests/visual-regression/scripts/build-cells.mjs:64-88` targets only `REFERENCE_BASENAMES` (Counter, SearchInput, Dropdown, TodoList, Modal, TreeNode, Card, CardHeader, ModalConsumer, WrapperModal) — TableDemo is NOT in that list, so leftover `.rozie.ts` artifacts from prior failed runs persist between builds (though the file is regenerated fresh on each run anyway, so cleanup alone doesn't fix the underlying emitter bug).
- The last successful Angular ModalConsumer baseline bless was at commit c877569 (2026-05-16 23:43:55), BEFORE d77ffeb introduced TableDemo on 2026-05-17 00:39. Every Angular VR build since d77ffeb has been silently broken.

**Proposed mitigation paths (orchestrator/user decision):**

1. **Recommended — Hotfix plan for the Angular emitter bug** (preferred per `feedback_no_false_equivalent_options.md` honest recommendation pattern):
   - One-line fix in `emitDecorator.ts:152-157` and `emitDecorator.ts:160-166`:
     ```ts
     const escapedTemplate = opts.template.replace(/`/g, '\\`').replace(/\${/g, '\\${');
     const indentedTemplate = escapedTemplate.split('\n').map(...).join('\n');
     ```
     (also escape `${` to avoid accidental template-literal interpolation; same fix for `stylesArrayBody`)
   - New unit test in `packages/targets/angular/src/__tests__/emitDecorator.test.ts` asserting backtick-escape for both `template` and `styles` blocks.
   - Re-bless any per-package + dist-parity fixtures that would change (likely zero — none of the current reference fixtures use backticks inside templates, so this is a defensive forward-compat fix).
   - Then re-run Plan 05 Task 2 (Docker PNG re-bless) — should complete cleanly.

2. **Alternative — Remove TableDemo.rozie + DropdownDemo.rozie from `examples/demos/`** until the emitter is fixed:
   - Lower-effort revert; loses dogfood demos.
   - Both files have a comment "Not registered with tests/visual-regression" but the Angular target's `prebuildExtraRoots: [examplesRoot]` walks them anyway.

3. **Alternative — Make Angular VR sub-build a soft-fail target** (add 'angular' back to `SOFT_FAIL_TARGETS` in `build-cells.mjs`):
   - The set is currently empty after Quick 260515-1y4 declared Angular hard-required; adding 'angular' back temporarily is allowed by the script comment but degrades the matrix's authoritative completeness.
   - Loses Angular VR coverage entirely; not recommended.

**Why this SUMMARY does NOT silently auto-apply path 1:** the plan explicitly states "Plan 05 does NOT touch emitter source. All emitter changes landed in Plans 01-04." Auto-applying an emitter fix in Plan 05 would violate that boundary AND would deliver an emitter fix without the unit-test coverage + fixture sweep that good hygiene demands. Better to defer to a dedicated hotfix plan with the right shape.
